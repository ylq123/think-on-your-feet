# 驻足思考 · 即兴演讲方法论

> "More paper means more bondage. With it you constipate eloquence, drive audiences to counting files on the ceiling."
> — Keith Spicer, *Winging It*

---

## 这是什么

一套让你在没有任何讲稿的情况下，也能有条理、有说服力、有魅力地张嘴就说的操作系统。

核心承诺：**你不是天生不会讲，你只是缺一套结构。结构有了，自由就有了。**

## Step 1: 大脑扫描图（3 分钟）

在你张嘴之前，唯一该做的事：**弄懂这个人的脑子里现在装着什么。**

拿出一张纸，画三列，3 分钟内填完：

```
| 他们知道什么 | 他们想要什么 | 他们怕什么 |
|-------------|-------------|-----------|
| 已有知识/偏见 | 希望/期待    | 恐惧/担忧  |
```

**操作方法**：
1. 不停笔，脑中冒出的任何信息都写下来
2. 圈出最显眼的关键词
3. 写下听众可能会问的问题（尤其是让你不舒服的）
4. **重点研究「恐惧」栏——恐惧永远比希望更有信息量**
5. 你的核心信息必须直接回应那个恐惧

**信息来源**：直接问上级/客户、根据经验判断、观察听众行为。

## Step 2: 核心信息设计

### 关键要点

= 对当前听众最重要的一件事。用听众的耳朵想，不以自己的喜好为准。

### 核心标题（3-7 词）

**选词法则**：

| 用 | 不用 |
|----|------|
| 熟悉词：「死亡」 | 生僻词：「过世」 |
| 具体词：「加薪」 | 抽象词：「报酬待遇方案」 |
| 短词：「现在」 | 长词：「在当前这个时候」 |
| 名词动词 | 形容词副词（「一位绅士」不说「举止温文尔雅的人士」） |
| 主动语态：「我认为」 | 被动语态：「据认为」 |

### 说服三步法

1. **集中** — 找到适合听众的主题：「我的听众想知道什么？」
2. **澄清** — 构建逻辑计划来讲授这个主题
3. **影响** — 监控听众反应，边说边调整

## Step 3: 逻辑结构设计

### 标准演讲格式

```
介绍(5%) → 主体三部分 → 结论(5%)
```

- **介绍**：第一句主题标题 → 第二句宣布三部分提纲 → （可选）暗示结论
- **结论**：重述三部分关键词 → 一字不差地呼应核心标题

### 「书立法则」

开头和结尾像书立——各不超过总时长 5%，但必须像刀刻的一样准。

> 好机长告诉目的地，不讲漏油故事。

### 选择逻辑计划

**三大法则**：
1. 形式服从功能——计划强化结论，不为结构而结构
2. 变换计划类型——主计划与子计划不同类
3. 变换片段数量——2-5 个片段之间变化，单调是催眠药

**6 种核心计划**：

| 计划 | 适用场景 | 示例结构 |
|------|---------|---------|
| **三角计划** | 并列维度 | 可行性/风险/建议 |
| **时钟计划** | 时间序列 | 过去/现在/未来 |
| **钟摆计划** | 对比论证 | 方案A → 方案B → 最佳方案C |
| **地点计划** | 空间维度 | 本地/全国/全球 |
| **变焦镜** | 层次切换 | 个人→部门→公司→行业 |
| **收益计划** | 利益罗列 | 对你/对团队/对公司 |

**双重测试**（选计划的硬门槛）：

- 🎬 **电影测试**：这个结构能拍出一部简单生动的电影吗？
- 🎯 **结论测试**：这个结构能导向我最想说的那句结论吗？
- 两个都是 No → 换计划。

### 复合技术

主计划类型 + 子计划类型交叉使用，产生多重说服效果。例如：三角计划（主）→ 子片段用钟摆和变焦镜交替。

### 克服「白纸瘫痪」

1. 把问题拆成可管理的小块
2. 对重要话题用 3-4 个不同计划探索新角度
3. 画图标出三个要点「桩子」
4. 从中选最有力量的一个

## Step 4: 脱稿即兴表达 · 三桩子法

核心原则：**左脑搭架，右脑跳舞。**

### 大脑分工

| 左脑（逻辑型） | 右脑（直觉型） |
|---------------|---------------|
| 逻辑/分析/线性/连贯 | 直觉/想象/形象/情感 |
| 搭建思想的「骨架」 | 提供「血肉」 |
| 负责控制和导航 | 负责自由发挥 |

### 执行步骤

```
左脑 → 三个桩子 → 宣布数字
  ↓
第一个桩子：左脑喊关键词 → 右脑展开细节（故事/例子/数据）
  ↓
第二个桩子：左脑喊关键词 → 右脑展开细节
  ↓
第三个桩子：左脑喊关键词 → 右脑展开细节
  ↓
左脑重述三个桩子 → 结束
```

**实例**：
> 「今天我从 *可行性*、*风险*、*建议* 三个维度来谈这个方案。首先，可行性——（右脑展开）……其次，风险——（右脑展开）……最后，建议——（右脑展开）。总结一下，可行性、风险、建议——我的建议是立刻启动。」

### 为什么有效

- 左脑的三桩子像数据库索引，帮你瞬间检索
- 宣布数字能清除无关情绪，给你安全网
- 右脑在每个桩子范围内自由飞翔——你不会跑题
- 你还能观察听众反应，中途调整

### 识别四大陷阱

| 陷阱 | 表现 | 纠正 |
|------|------|------|
| **坏掉的铁路** | 偏离话题，逻辑横七竖八 | 回到桩子，确保逻辑直线 |
| **起伏的山峦** | 逻辑上下起伏 | 提炼核心概念，保持平稳 |
| **过山车** | 又急又快，用速度补空洞 | 慢下来，用停顿和结构替代语速 |
| **沙漠迷踪** | 滔滔不绝地旁征博引 | 时刻用「结论先行」点明方向 |

## Step 5: Q&A 应对 · AIR 法则

### A — 认可 (Acknowledge)

在讲任何道理之前，先接住对方的情绪。

**认可个人**：「我明白您为何对此关注。」（避免过度使用「这是个好问题」）

**认可事实**：「您说得很有道理，因为您提到……」

**认可感受**：「我看得出来，你对此感受强烈。」——情绪不散，逻辑不进。

### I — 询问 (Inquire)

**澄清意思**：「你能否重复一遍那个问题？」「你具体指的是什么？」

**询问例子**：「你是否能举个例，说明你在哪里碰到了问题？」

**封闭式提问**：听众焦虑/生气时，用「是/否」类问题控制节奏、收集信息。

### R — 调整 (Adjust)

**关键词引导**：「首先，让我们聚焦质量标准……如果时间允许，再讨论成本。」

**改变层次**：
- 上升 → 拉到更宽泛的层次
- 下降 → 聚焦到具体细节
- 偏向 → 只处理你能控制的部分

**言归正传**：发问者跑题时——「由于今天重点是 X，我们能否会后讨论你的 Y？」

**转化**：将负面提问转为正面回答。

## 高级表达框架

### 框架一：变焦镜

通过视野的动态切换引导听众。

**拉远**（扩大视野）：细节 → 大局 → 说明意义
- 适用：处理敏感信息、证明决策合理性

**拉近**（聚焦细节）：全局 → 具体事例 → 推翻笼统假定
- 适用：反驳一概而论、具体化宽泛认识

**动态切换**：小→中→大 或 大→中→小 的同心圆推进。

### 框架二：钟摆辩证

**激进形式**（推翻两端，捍卫中间）：
1. 极端 A 及其负面后果
2. 极端 B 及其负面后果
3. 中间立场，证明其明智

**温和形式**（调和两端，走向共识）：
1. 承认 A 的合理性
2. 承认 B 的合理性
3. 综合为双赢方案

**话术**：「从一方面看……」「反过来看……」「平心而论……」

### 框架三：图景辅助

用类比或实例，把抽象概念转化为听众头脑中生动的画面。

**类比**：源域（听众熟悉）→ 目标域（要解释的）→ 建立相似点
- 关键：类比物必须来自听众的世界，不是你的世界。

**实例**：用包含确切数据和生动细节的真实案例替代抽象解释。

## 快速参考卡

### 演讲前检查清单

- [ ] 大脑扫描完成（他们知道/想要/怕什么）
- [ ] 核心标题 ≤ 7 词（主动语态、具体名词动词）
- [ ] 三桩子选好（关键词能"挂"住思想）
- [ ] 通过电影测试 + 结论测试
- [ ] 准备好三个最尖锐的问题和最诚实的回答
- [ ] 开头结尾已打磨（≤ 各 5% 时长）

### 被突然点名时的闪电流程

1. **别张嘴** — 心里闪电般立三个桩子（3 秒）
2. **宣布数字** — 「我从 X、Y、Z 三个方面来谈」（2 秒）
3. **切换右脑** — 一个桩子一个桩子展开
4. **回到左脑** — 呼应收尾

### 被愤怒质问时的三步走

1. **认可** — 「我看得出来，你对此感受强烈」
2. **询问** — 「你能具体说说，你觉得问题出在哪？」
3. **调整** — 选择关键词引导回答、改变层次、转化负面

## 关键原则速记

| 原则 | 一句话 |
|------|--------|
| 沟通 = 教学 | 让人改变想法的最好办法是教会他们，不是战胜他们 |
| 左脑搭架，右脑跳舞 | 最好的即兴表达是有结构框架下的自由发挥 |
| 越准备，越不需要稿 | 准备充分了，稿子就是枷锁 |
| 简单即武器 | 奶奶听不懂不是因为你聪明，是因为你不会说 |
| 先接情绪，再讲逻辑 | AIR：认可→询问→调整 |
| 扫描再张嘴 | 恐惧永远比希望更有信息量 |
| 电影 + 结论 | 两个测试都过不了的计划不要用 |
| 书立法则 | 开头结尾各 ≤ 5%，但必须刀刻般精准 |

## 诚实边界

- 本方法论主要面向商务演讲、工作汇报、媒体采访等「常规高压」场景，不适用于战地报道、刑事审讯等极端场景
- 三桩子法需要练习——第一次用的时候左脑可能背叛你，像学钢琴前三周手指头不听使唤
- 钟摆辩证不适用于只有一方绝对正确的原则性/底线问题
- 方法论源语言为英语，部分修辞技巧（如 3-7 词标题）在中文语境下可能需要调整（中文标题可以更精炼）
- 过度依赖结构可能让讲话显得「公式化」——结构是脚手架，不是房子本身，熟练后应该内化而非背诵
- 基于 Keith Spicer *Think on Your Feet* (1986) 和 *Winging It* (1982) 提炼
